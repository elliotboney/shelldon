- (decision) A. SCHEDULER: core-resident scheduler of named multi-cadence jobs (interval/cron/idle-triggered) replaces v1's single heartbeat loop. Jobs tagged by COST TIER — reflex jobs (mood drift, blink) run in-core, no LLM, cheap CPU; turn jobs (reflection, dreaming, proactive messages) cost a fork+LLM, are few, cooldown-gated, on a daily CREDIT/turn BUDGET. Battery-aware: reads PiSugar2 power state, stretches cadences / skips non-essential LLM turns on battery or low charge, livelier when plugged in. Incoming messages/events bypass the scheduler (immediate). Heartbeat is just one job, not the engine.
- (decision) B. DREAMING: a scheduled introspective WORKER TURN (idle / context-pressure / cadence triggered) that consolidates recent conversation history, classifies + promotes captured learnings, and prunes. Reuses the normal worker/broker path; proposes memory-ops; core writes (single-writer). Not a separate subsystem, not a v1-style direct module.
- (decision) C. SELF-IMPROVING LEARNING (light): Capture (hot path) — worker proposes capture_learning(observation, pattern_key?) memory-op during normal turns; core writes a row to sqlite learnings table (dedup by pattern_key, increment recurrence_count, status pending); cheap, no extra LLM. Promote (dream cycle) — classify pending learnings, promote durable/high-value ones (LLM judges by impact+recurrence, NOT a rigid count) into curated markdown (about.md/facts); sensitive ones route to vault (broker-gated); prune the rest. LIGHT: no ERRORS/FEATURE_REQUESTS taxonomy, no promotion-to-CLAUDE.md / skill-extraction. sqlite = raw+queryable (history + learnings), markdown = curated+durable; dream cycle bridges them.
