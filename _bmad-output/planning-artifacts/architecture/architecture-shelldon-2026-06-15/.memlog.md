- (decision) TOPOLOGY (AD candidate): multi-process. Broker is a separate trust-boundary process = sole holder of creds + sole network egress to models/tools. Fork-server parent pre-warms LLM LIBS only (never creds), os.fork()s a worker per turn; worker assembles prompt with warm libs but the authenticated model/tool call is proxied to the broker over IPC, so workers get warm-start AND never hold creds. Binds: all. Prevents: forked workers inheriting cred pages via COW; an injected tool call reaching credentials. Chosen over single-process (soft isolation) and broker-does-everything (fat bottleneck, cold workers).

- (decision) STACK RATIFIED: Python/CPython. Decisive: the fork-server warm-start+RAM-free bet requires os.fork() COW, which Node/V8 cannot do (child_process.fork spawns a fresh V8, no memory fork; V8 heap not fork-safe). Plus Pi hardware is Python-first (Waveshare, gpiozero/spidev, bleak BLE, PiSugar) and v1 is Python. Agrees with spec non-goal (Node excluded).
- (decision) BUS (AD candidate): transport = Unix domain sockets carrying length-prefixed msgspec bytes (stdlib, asyncio-native, no deps, low RAM). Topology = hub-routed: core hosts the bus and is the router; broker/display are resident clients, workers are connect-do-die clients spawned by the fork-server. Core sees every Job envelope = single audit/trace point; accepted that core sits on the turn's hot path. Job envelopes carry NO creds; broker injects creds internally. Binds: all inter-component comms. Prevents: incompatible message framing/addressing; scattered point-to-point channels with ephemeral workers.
- (decision) DATA OWNERSHIP (AD candidate): core is sole owner + sole writer of the personality-state struct AND the sqlite store. Reflexes mutate the struct directly (in-process). Workers NEVER write state/memory — a turn returns a Result envelope with PROPOSED deltas (mood shift, new memories); core validates + applies. Workers read memory via the bus (core serves a snapshot). Display never reads shared memory; core PUSHES a state-snapshot envelope on change, display renders from it. sqlite writes batched in core (write-behind) to spare the SD card. Binds: all state/memory access. Prevents: write races, two writers corrupting sqlite, an injected worker corrupting the soul or DB.

- (decision) PLUGIN CONTRACT (AD candidate, CAP-7): a plugin is a BUS CLIENT speaking only Envelope — never imports/touches core internals (the import-linter enforces the same rule from core's side). Bounded RAM (one process regardless of plugin count); host-level crash isolation; plugins isolated from core by process boundary. A crashed plugin kills sensing, not the soul. Chosen over process-per-plugin (RAM cost on 512MB).
- (decision) ARBITER (AD candidate, in core): single component owns reflex-vs-turn decision + turn governance. (1) Turn concurrency = strict ONE worker turn in flight at a time (each fork holds warm LLM libs; concurrent forks blow 512MB). (2) Events during a turn COALESCE into the next turn's context, never queue a backlog of turns (prevents poke-stampede + cost spikes). (3) Proactivity (CAP-4) gated by a minimum-interval COOLDOWN (daily turn budget deferred unless cost-paranoid); reflexes carry the in-between. (4) Degradation: provider-chain exhaustion -> arbiter falls back to a reflex behavior so the pet never freezes. Binds: CAP-1/2/4 + all turn invocation.
