## What went well

1. **The Epic 3 significant-discovery got resolved exactly as committed.** The fire-and-forget topology that blocked memory-ops was decided up-front (Epic 3 action #2 → 4.2 split → 4.5 wire), and the **4.2/4.5 apply-half / wire-half split paid off** — 3.4 then rode the finished wire as a 3-edit story with a clean run.
2. **Proactive `$HOME` isolation became reflex.** Epic 3's #1 recurring pain (real-`$HOME` writes caught in verify *twice*) is dead — **every** Epic 4 story extended the conftest fixture *in the same change*. Epic 3 action #3 fully landed.
3. **Idiom reuse compounded.** Atomic-write (temp→fsync→`os.replace`), fail-soft best-effort guards (mirroring `_record_turn`), whole-reject validation — propagated verbatim across history, memory, vault, and prompt. Building idioms, not re-solving.
4. **The security story gated honestly.** 4.3's real `PermissionError`-from-kernel test is `skipif` Linux+root and was **never faked green** on macOS; mechanism + perms + broker-authority still tested everywhere.
5. **CAP-6 is real.** The epic's headline — memory *shapes the reply*, not just stored — is proven by a fact from turn 1 reaching turn 2's prompt.

## Challenges / themes (systemic, no blame)

1. **Longer-lived workers exposed a hidden wedge.** 4.5's reshape (worker now awaits completion) surfaced: a silent broker crash gives **no EOF** on the dropped hub connection → worker blocks **forever**, locking the ≤1-worker bound. Required an unplanned `wait_for` timeout backstop *not in the story text*. New topology = new failure modes.
2. **The same review-finding classes still recur.** Epic 3 action #5 (recurring-review self-checklist) didn't prevent them: missing exception guards on best-effort paths (4.1, 4.4, 4.5), and **weak/false-positive tests** (4.4's CAP-6 false positive masked by the system-instruction example; 4.5's AC2 ordering unverified; 3.4's missing rejection tests). Reviews keep catching them; the gate doesn't.
3. **4.4 was "previously mis-marked done."** A story tracked complete that wasn't (incomplete isolation + error-handling sweep). A done-definition gap.
4. **Timeout asymmetry now lives unguarded** — 120s worker / 90s `worker_in_flight` / 30s core degrade. Documented as deferred; a real freeze window.
5. **Atomic-write helper triplicated** (faces.py, state.py, memory.py). Acknowledged debt; shared extraction deferred again.
