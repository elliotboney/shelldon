## What went well

1. **The provider abstraction absorbed scope instead of fracturing under it.** A native `GeminiProvider` (`google-genai`) was built, then *deleted* when the owner observed Gemini fits the OpenAI-compatible endpoint. The epic ended with **9 providers (glm/claude/ollama/openai/openrouter/groq/cerebras/nvidia/mistral/github/gemini) and zero per-provider code** — data-driven rows in a `_OPENAI_COMPAT` table. Adding a provider is a config line, exactly the AC promise.
2. **Seam-first design paid off a second time.** 2.1 documented the precise spot where 2.2 would add chain iteration (`run_broker` holds the chain, runs `chain[0]`). 2.2 added `handle_job_chain` and flipped one call site — no rework. 2.3 then degraded on the terminal failure 2.2 produced, again no rework.
3. **2.3 stayed disciplined.** The degrade path already existed (1.8) and 2.2 produced the terminal failure, so 2.3 was proof + de-placeholder: 3 end-to-end tests + comment edits + two one-line `is_idle` properties. It explicitly did **not** build a second degrade mechanism, a resident reflex loop, or real expressions (correctly deferred to Epic 3).
4. **Mechanical invariants carried the load again.** Import-linter KEPT every story with zero drift; every LLM SDK (`anthropic`, `openai`) stayed quarantined in `broker/`.
5. **Reviews caught real defects and rejected false positives with evidence.** Real catches: Ollama double-`/v1` corruption, case-insensitive preset lookup, `writer.wait_closed()` missing on fast reconnect, the `str(sdk_exc)` credential leak. Evidence-based rejections: the `run_broker` `finally` NameError (the `connect()` is outside the `try`), and the "permanent errors silently eaten" claim (they're already logged at WARNING).
6. **Deferred-work discipline held.** Every punt across all three stories is in `deferred-work.md` with a reason and a revisit trigger.

## What was hard (recurring themes)

1. **Timing-anchored test flakiness — now two epics running.** Flagged in Epic 1 (action #6), recurred in 2.3 (the 2s degrade bound). The pattern is band-aid (bump the bound) rather than structural fix (state-predicate assertions + a diagnostic `_await` that dumps registry/turn state on timeout). The `_await` poor-diagnostics deferral is going stale.
2. **Protocol changes ripple through every test fake.** Adding `name: str` to the `LLMProvider` Protocol forced touching `_OK`/`OkProvider`/`GatedProvider`/`AlwaysTransientProvider` across 5 files. The reviewer flagged this as metadata coupling (deferred #8) — a `(name, provider)` tuple/NamedTuple at the chain layer would decouple the audit label from the provider contract.
3. **Env-coupled tests are fragile.** An ambient `GEMINI_API_KEY` in the dev shell caused a key-gated live test to run during the normal suite (2.1). Fixed with `addopts = "-m 'not live'"` (live is now opt-in), but it shows how environment state leaks into the suite.
4. **Test code isn't held to the same DRY/clarity bar as prod.** Duplicate `_Collector` across two broker test files, a hardcoded `primary.calls == 2` retry-count assert, and a `conftest` docstring overstating reconnect-backoff coverage — all real, all deferred (2.3 review).
5. **Mid-story scope expansion via owner requests.** 2.1 absorbed Gemini + 6 free-tier providers *during* the story. It worked precisely because the abstraction is config-driven and each addition was logged as a Change Log delta — but it's a pattern to keep visible (additions = explicit scope deltas, not silent growth).

## Epic 3 readiness assessment (next: A Pet That Feels Alive)

Epic 3 = 3.1 persistent personality-state struct → 3.2 resident reflex loop → 3.3 expressive face via the display compositor.

- **Clean seams already exist.** 2.3 deliberately left `FACE_DEGRADED` a placeholder token and the "reflex behavior" as just the degrade ack — 3.3 replaces the tokens with real expressions, 3.2 builds the live reflex loop. The display has a `StubRenderer` seam (1.7) for the 3.3 compositor; core's single-writer loop is the home for 3.1's state writer.
- **New territory (the real work):**
  - **3.1 — first persistent state with checkpoint/restore + SD-wear discipline.** No sqlite/atomic-write/periodic-checkpoint code exists yet. New patterns.
  - **3.2 — first in-core periodic tick.** No scheduler yet (Epic 5), so the tick must be built standalone but *structured to be subsumed* as a cost-tier reflex job later (the AC says so — no forward dependency).
  - **3.3 — first real E-Ink expression rendering.** The display has only ever run with `StubRenderer`; no real panel (or faithful emulator) has rendered. This is the **concentrated unknown** of Epic 3 — the mirror of Epic 2's "no real provider had ever run," which the 2.1 live-smoke spike retired.
- **Cross-check 2.3 left for 3.2:** confirm the degrade ack coexists with a live reflex loop (no latch; reflexes carry the offline in-between). 2.3 stands alone today; 3.2 is where they meet.
- **No significant discoveries** invalidate the Epic 3 plan. Direction is sound.

## Key takeaways

1. **A config-driven abstraction turns scope requests into config lines** — Gemini + 6 providers folded into 2.1 with zero new code because the seam was right. Keep building seams that absorb rather than fracture.
2. **Seam-first continues to eliminate integration rework** — 2.1→2.2→2.3 chained with no rework because each story documented the next story's insertion point.
3. **Recurring test-flakiness is the one lesson not being learned** — it's been band-aided twice. Epic 3 should fix it structurally, not bump another bound.
4. **"Epic 2 will handle it" debt needs an owner, not an epic** — the arbiter/fork resilience items survived Epic 2 because they were assigned to an epic whose actual scope was the broker. Assign debt to a story/AC, not an epic name.
