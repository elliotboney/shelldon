## What went well

1. **CAP-7 held mechanically, every story.** Import-linter stayed 3 KEPT / 0 broken across 7.1–7.5; `uv sync --locked` 0 new deps each story; the schema never bumped. 7.3 and 7.4 *assert* `git status -- shelldon/core/` empty in their gate — the "behavioral capability with no core touch" thesis was demonstrated, not asserted. Then 7.5 touched core **deliberately and minimally**.
2. **Clean 5-seam progression, each forcing the next.** contract+host (7.1) → core-emits/plugin-consumes (7.2) → plugin-draws (7.3) → plugin-emits (7.4) → plugin→core affect (7.5). Every story consumed the prior one's seam and **resolved its icebox** — 7.2 closed 7.1's read-loop limit, 7.3 closed the FACE-claim gap, 7.4 consumed the `emits` field declared-but-unused since 7.1. No orphaned scaffolding.
3. **Both off-plan stories were deliberate, spec'd, and owner-gated — nothing slipped without a story slot.** 7.0 (retro-born prep, same lineage as 5.0) and 7.5 (brainstorm-born). The Epic 4 lesson ("chores with no story slot don't happen") held in the inverse: even *unplanned* scope got a real slot. 7.5 especially — a mid-epic scope expansion was handled by spawning a new spec'd story rather than smuggling a core change into 7.4, which preserved 7.4's zero-core boundary as a clean CAP-3 proof.
4. **Patch trend stabilized to clean.** Patches peaked at 7.3 (7) — the overloaded first-real-plugin where the draw seam + atomic state I/O + harness all landed at once — then collapsed to 1 (7.4) and **0 (7.5)**. The seam matured over the epic.
5. **The 7.0 prep refactor paid off silently.** No story 7.1–7.5 reported `runtime.py` coupling pain — exactly what the extract was for. "Harden the seam before building on it" (the 5.0/7.0 discipline) worked again.
6. **+89 tests, 0 deps, 0 incidents, contracts KEPT, 0 mis-marked-done** — the clean-delivery streak (every epic since 1) continues.

## Challenges / themes (systemic, no blame)

1. **A dev-introduced regression class recurred: critical-section / slot-release.** 7.0 (arbiter slot leaks if `apply_patch` raises on the error path) and 7.2 (the turn slot is held forever if a backpressured plugin-host `drain()` suspends *before* `arbiter.complete()`). Both subtle (only bite under a raise/suspend), both the same shape: a fail-soft side-effect placed wrong relative to the guard that frees a ≤1 slot. This is **the** dev-introduced bug class of the epic, and it was NOT on the checklist (now added).
2. **Bus-client resilience was deferred three times and never paid down.** `OSError`-escape from the read loop / no drain-backpressure bound / no `connect()` retry surfaced in 7.1, 7.2, AND 7.4 — each iceboxed as "pre-existing whole-bus-client class." It accreted into `bus-client-oserror-and-drain-resilience` but stayed unpaid all epic. A real consumer-wedge or a reconnect would expose it.
3. **The checklist exists but wasn't self-applied before review.** The OSError-resilience class it explicitly tracks still reached *review* in 7.1, 7.3, and 7.4 (caught by the reviewer, not the dev pre-flip) — the exact failure mode the checklist was created to prevent. The checklist is good; the *habit of running it before flipping to review* is the gap.
4. **7.3 was overloaded** (7 patches + 3 decisions in one story) — it bundled the draw seam + atomic state I/O + the first real plugin + lazy-load isolation. The "first real consumer" story carried disproportionate review weight; consider splitting the seam from its first consumer next time.
5. **Bus-e2e harness gotchas cost a TDD detour twice** — 7.2 (a no-subscriber WARNING flood tripped `tracemalloc`'s soak measurement → fixed at source, severity-correctness) and 7.5 (a `_`-discarded `StreamWriter` was GC'd and deregistered the actor mid-test). Both are harness-lifetime traps, not product bugs; both are worth a "keep the writer referenced / watch log severity in the soak" harness note.

## 🚨 Significant-discovery check → what comes next

**The strategic fork is unchanged from Epic 6 — and now overdue by an epic.** The honest next move is the one deferred twice: **run the core against a live LLM before extending or deploying further.** Owner decision this retro: **the binding next move is the live-LLM smoke (Epic 6 action #1).** Everything built since Epic 4 — turns, memory ops, the dream, and now plugin events shaping mood — assumes a real model emits the right ops; that assumption is still 100% untested.
