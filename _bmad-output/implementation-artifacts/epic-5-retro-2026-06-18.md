## What went well

1. **The Epic 4 retro's headline fix landed exactly.** "Chores without a story slot don't happen" → Story **5.0 gave resilience its own slot** and it could not be skipped under feature pressure: it absorbed the 4.5 worker-wedge, the 120/90/30s timeout asymmetry, and the floating Epic 2 #3 arbiter/fork debt, then **gated 5.1**. The `dev-loop-checklist.md` was applied on every story (Epic 4 action #2/#3 fully paid off).
2. **"Build the mechanism ahead of the consumer" became the epic's spine.** `Job.cost` (5.2), `Job.essential` (5.3), `Job.prompt_builder` + `history_owner_text` (5.4) — each a tested seam built before its real user exists. Epic 6's dream turn just *sets* them; near-zero rework predicted.
3. **Gate layering stayed surgical.** 5.3 battery is an OUTER gate over 5.2's **untouched** `_dispatch_turn_job`; 5.4 added **no** gating at all and rode both. Each story explicitly listed what it did NOT touch — and held to it.
4. **Owner decisions front-loaded, every story.** Each feature story locked 2–4 owner decisions up front (budget=12/cooldown=30min; 3-tier/eased×3/threshold 20%; Idle-trigger/mood→prompt/thought-framing/1hr). No mid-build thrash, no re-litigation.
5. **The done-gate held.** The 4.4 "previously mis-marked done" gap did **not** recur — every story flipped to review/done with the isolation + error-handling sweep + value-asserting tests present.
6. **Parallel subagent worked (5.4).** The isolated pure module (`core/proactive.py` + its tests) was built by a background subagent against a fixed signature while the coupled `scheduler`/`runtime` wiring proceeded — clean file-ownership split, zero contention.

## Challenges / themes (systemic, no blame)

1. **The recurring review class is now "tick/lifecycle resilience edge cases" — and it still reaches review.** Every story's review caught the *same shape*: one input class that slips past the guard the dev did write — tz-naive AND future timestamps (5.2/5.4), unguarded power read (5.3), whitespace-only / NaN / `cost<1` / reflex-with-no-`run` (5.2/5.3/5.4). The checklist catches the common case; the **long-tail input edge** still lands in review. Trend is down (8→2→3) but the class persists.
2. **The whole epic lives in two files.** `core/runtime.py` (~560 lines now) and `core/scheduler.py` carry 5.0–5.4 intermingled — to the point that **per-story commits were impossible** (5.2–5.4 shipped as one commit). `Core` is now doing turn-lifecycle + scheduler-loop + dispatch + reflex + proactive. Coupling is climbing.
3. **5.4's proactive prompt rides the owner-message slot** (no `worker/prompt.py` change) — a deliberate MVP shortcut. Untested against a real model; real LLM runs may need a dedicated proactive prompt section.
4. **Still no live-LLM and no-hardware verification.** Battery backoff is proven against an injected stub (real PiSugar2 = Epic 7); proactive is proven with no real model. The autonomy line is **mechanism-proven, not field-proven.**
5. **Atomic-write helper still triplicated** (faces/state/memory) — carried from Epic 4; the "4th writer" trigger wasn't hit, so the icebox entry stands.
