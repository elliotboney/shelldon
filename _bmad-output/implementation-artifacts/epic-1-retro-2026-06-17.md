## Epic summary

- **Stories:** 9/9 done (1.1 scaffold → 1.9 endurance), every story code-reviewed.
- **Tests:** 89 passing / 2 skipped (Linux-gated real-fork tests). Grew from an empty M0 harness to full cross-actor integration + soak.
- **Invariants:** both import-linter contracts — `core is LLM-free (AD-1)` and `transport holds no model/tool creds (AD-13/NFR9)` — **KEPT for the entire epic**. Mechanical, never broke.
- **Production incidents:** 0 (pre-deployment; target is the Pi Zero 2W).
- **Outcome:** end-to-end turn works (message in → LLM reply → face reacts) and NFR2 (no memory growth) is proven, not just designed for.

## What went well

1. **The test ladder worked as designed.** Per-story isolation tests (fakes/stubs) → 1.8 *confirms* the wiring → 1.9 *proves* endurance. 1.8 integrated all five actors with **no rework** because each piece was independently tested first.
2. **Injected seams made everything testable without hardware.** Fork-spawn seam (1.5), inbound/outbound seams (1.6), renderer seam (1.7), provider protocol (1.4). The whole system runs cross-platform in-process with no real `os.fork()` and no panel — this is why 1.8/1.9 run green on macOS.
3. **Reviews caught real defects, not nits.** 1.8 P1 (spawn-failure left the fence open + arbiter stuck) and 1.9's measurement flaws (N-dependent heap bound; a suggested monotonic check that false-fails on flat RSS) were genuine pre-merge catches.
4. **Deferred-work discipline.** Every review punt is logged in `deferred-work.md` with a reason and a revisit trigger. A mid-epic "deferred-item sweep" even burned down cheap coverage gaps in parallel.

## What was hard (recurring themes)

1. **asyncio teardown/cancellation under Python 3.13 is the sharp edge.** The `BusServer.stop()` hang (`Server.wait_closed()` blocks on idle clients parked in `read_frame`) was subtle and cost real debugging. The fix (track + cancel handler tasks, close listener last) is the most intricate code in the tree.
2. **Test synchronization fragility.** Repeated `asyncio.sleep(0.05)` polling (flagged from 1.3 onward); improved with the `_await` predicate helper, but timing-anchored assertions (1.8/1.9 timeout tests) remain a flake risk.
3. **Two ≤1 guards stayed independent until 1.8 (planned).** The arbiter policy gate and the ForkServer mechanical guard were only wired at integration — which is when a real race (timeout + catch-up → `WorkerBusyError`) became visible. Caught by 1.8 review (P1); the dropped catch-up prompt is deferred to Epic 2.
4. **Writing a correct soak is its own skill.** 1.9 took two iterations to get an N-independent, non-flaky memory measurement — harder than the system it tests.

## Epic 2 readiness assessment

- **Foundations are clean.** The `LLMProvider` protocol (1.4) and the core degrade path (1.8 AC3) already exist — 2.1 generalizes the provider into an ordered chain; 2.3 wires the broker's terminal failure to the existing degrade. Extend, don't invent.
- **The one real unknown:** no real provider has ever run — Epic 1 used fakes throughout. 2.1 builds the first live Anthropic-format adapter (GLM-5.2 via Z.ai's Anthropic-compatible endpoint); first exercise of creds + network. The `anthropic==0.109.2` SDK is pinned but never called.
- **2.2's load AC is leverage, not new work.** "Provider faults under sustained load → fallback holds, no crash/hang/leak" is **1.9's soak harness + 1.8's fault fakes** (`GatedProvider`, `AlwaysTransientProvider`) reused with fault injection. No replan needed.
- **No significant discoveries** that invalidate the Epic 2 plan. Direction is sound.

2. **Deferred resilience cluster:** **Fold into Epic 2 stories.** Each item becomes acceptance criteria in the story that needs it (backoff/reconnect/timeout/reap-watchdog → 2.2; degrade wiring → 2.3) rather than floating debt.
